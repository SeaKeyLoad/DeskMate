# 🎯 行为坐标标注工具 v2.2 - 操作手册

本手册专为使用「行为坐标标注工具 v2.2」进行 `behavior_rules.json` 数据打标和规则生成而编写。该工具支持通过屏幕框选和 UI 自动化探测，快速生成高精度的控件定位与行为规则 JSON 数据。

---

## 💻 环境准备与启动

在开始打标之前，请确保您的 Python 环境已安装必要的依赖。

**推荐安装项（关键）：**
为了能够自动提取深层 UI 控件信息（如 `ControlType`、`AutomationId`），强烈建议安装 `uiautomation` 库。
如果未安装，工具将降级为仅使用窗口相对坐标计算。

**启动工具：**
运行 Python 脚本 `LabelerTool.py`，进入标注工具主界面。

---

## 📝 核心打标流程

数据打标分为三个主要步骤和一个导出步骤，请按界面顺序依次操作。

### 📋 第一步：选择目标窗口
此步骤用于锁定你要操作的具体应用程序和窗口，为后续的坐标基准（Baseline）提供参考。

1. **刷新进程列表：** 点击 **[🔄 刷新]** 按钮，获取当前系统中所有正在运行的进程。
2. **选择进程：** 在“选择进程”下拉菜单中，找到并选择你要打标的目标程序（例如 `WeChat.exe` 或 `chrome.exe`）。
3. **选择窗口：** 在“选择窗口”下拉菜单中，选择该进程下对应的具体窗口。
   * *注：系统会自动读取并显示该窗口的当前尺寸（宽 × 高）和进程名。*

### 🎯 第二步：框选目标 & 自动提取控件信息
此步骤用于获取目标按钮或区域的精准坐标以及 UI 控件特征。

1. **开始框选：** 点击 **[🖱️ 框选目标区域]**。
   * 此时主窗口会隐藏，屏幕变暗并出现十字光标。
   * 按住鼠标左键，**拖动框选**你需要操作的目标区域（如一个按钮或输入框）。
   * 松开鼠标完成框选；如果想取消，请按键盘的 `ESC` 键。
2. **确认提取结果：** 框选完成后，主界面会恢复显示。系统会自动计算并填入以下数据：
   * **坐标数据 (Baseline)：** 绝对坐标、宽高、相对窗口的百分比坐标。
   * **控件特征 (UI探针)：** 如果目标支持 UIAutomation，系统会自动抓取 `ControlType`、`AutomationId`、`Name` 和 `ClassName`。
   * *注：系统会根据你框选的大小，自动为你计算合适的“像素容差”和“百分比容差”。*
3. **重新提取（可选）：** 如果觉得刚才抓取的数据不准，可以点击 **[🔍 重新提取]** 重新抓取当前中心点的控件信息。

### ⚙️ 第三步：配置行为规则
结合上一步提取的数据，为当前目标赋予业务逻辑和定位规则。

1. **意图标签 (Intent Tag)：** 描述该操作的目的，例如：“点击登录按钮”、“输入账号”。这会作为 JSON 中的 `intent_tag` 字段。
2. **匹配模式 (Mode)：**
   * `anchor`：锚点模式（通常配合像素容差使用）。
   * `adaptive`：自适应模式（通常配合百分比容差使用，适合窗口大小会变化的场景）。
3. **定位策略 (Strategy)：** 系统会根据点击位置自动推断（如 `bottom_center`、`top_left`），你也可以手动下拉修改，指示脚本在执行时从哪个方向寻找目标。
4. **容差调整 (Tolerance)：** 确认自动生成的“像素容差”和“百分比容差”是否合适，必要时手动微调。
5. **控件限制 (可选)：** 如果自动提取到了控件特征，`控件类型限制` 和 `ID限制` 会自动填入。这能大幅提升匹配的准确率。

## 💾 生成与导出 JSON 数据

完成上述三步配置后，即可生成规则并导出为 `behavior_rules.json` 文件。

### 1. 预览与校验
* 点击 **[📋 预览并生成 JSON]**。
* 系统会弹出一个独立的预览窗口，展示当前配置生成的 JSON 源码。
* 你可以滚动查看，确认无误后，点击 **[📋 复制 JSON]** 直接复制到剪贴板，或者关闭窗口继续下一步。

### 2. 添加到列表（批量打标）
* 确认规则无误后，点击 **[💾 添加到列表]**。
* 当前规则会被保存到下方 **[📝 已生成的规则列表]** 中。
* **重复执行第一至第三步**，将所有需要的行为动作逐一添加到列表中。

### 3. 导出规则文件
* 当所有行为都打标并添加到列表后，点击右侧的 **[📁 导出所有]**。
* 在弹出的保存窗口中选择路径，系统默认会生成类似 `behavior_rules_20231024_120000.json` 的文件。
* 导出的文件即为完整的结构化打标数据，可直接供后续的自动化脚本或模型读取使用。

## 🔍 附录：输出 JSON 数据结构说明

成功导出的 `behavior_rules.json` 文件是一个数组，包含多个规则对象。单条规则的数据结构如下，供理解参考：

```json
[
  {
    "match": {
      "Process": "chrome.exe",
      "ControlTypeName": "ButtonControl",
      "AutomationId": "login-btn"
    },
    "locate_rule": {
      "mode": "anchor",
      "strategy": "bottom_center",
      "baseline": {
        "W": 1920,
        "H": 1080,
        "X": 960,
        "Y": 540
      },
      "tolerance": 45
    },
    "intent_tag": "点击登录按钮"
  }
]
```
